Quick Start — first time setup

1
Connect to your Pi

Open this page from the Pi's address in your browser: http://<pi-ip>:8765/. The connection dot at the top turns green when the backend is reachable. If it stays amber, check the IP and port and click Connect.

2
Set your music folder — Libraries tab

Go to Libraries. Type the full path to your music folder in Main library folder (e.g. /media/usb/Music). If you have a USB drive plugged in it will appear in the USB / Mounted Drives list — click it and press Use as Main Library to fill the path automatically. Then click Save Settings, then Scan Main Library.

3
Test playback — Main tab

Click ▶ Play Random. The Now Playing bar at the top should update with a track name and the VU bars should animate. If nothing plays, check the Logs tab for errors.

4
Set up a schedule (optional) — Scheduler tab

If you want the radio to automatically switch between different show folders at set times of day, set a Start time, add show blocks, and click Save Scheduler Settings. The schedule starts automatically at the configured time each day.

Connection bar

The bar at the very top lets you connect to your Pi from any device on the same network.

  • Pi address — the local IP of your Raspberry Pi (e.g. 192.168.1.100). Find it by running hostname -I on the Pi.
  • Port — usually 8765. If the backend couldn't claim that port it moves to 8766, 8767 etc. — the correct port is always shown in backend.out.
  • Connect — press after changing either field to reconnect immediately.
  • The dot shows Connected, Connecting…, or Offline. When offline the UI retries automatically every few seconds.
✎ The page remembers your last IP and port. If you always open it from http://<pi-ip>:<port>/ the connection is set automatically.

Main tab

The home screen. The Now Playing bar shows the current track and backend status at all times.

  • ▶ Play Random — picks a random track from your main library and starts playing immediately.
  • ■ Stop — stops playback. The scheduler will still start the next scheduled block at its configured time.
  • ⟳ Refresh Status — polls the backend for the latest status immediately (it also polls automatically every 5 seconds).
  • Quit Backend — shuts down the backend process entirely. You will need to restart it on the Pi.

The shortcut buttons (Libraries, Scheduler, etc.) are just quick links to the other tabs.

Libraries tab

Folders

  • Main library folder — the root folder the Pi scans for music to play during random/fill playback. Type a full path like /media/usb/Music.
  • Show library parent folder — a folder whose subfolders each become a separate show. For example if this is set to /media/usb/Shows, then /media/usb/Shows/Jazz and /media/usb/Shows/News each appear as individual shows in the Scheduler.

Volume

Drag the slider to set the master playback volume (0–100). Takes effect immediately on Save.

Fade transitions

  • Enable fade transitions — when ticked, tracks fade out before stopping and fade in when starting. Applies to show handoffs and commercial breaks.
  • Fade out / Fade in seconds — duration of each fade. Set to 0 for an instant cut.

USB / Mounted Drives

Lists drives the Pi has detected. Click a drive to select it, then use Use as Main Library or Use as Show Library to set that drive's root path as the corresponding folder. Press ⟳ Refresh to re-detect drives after plugging one in.

Scanning

  • Scan Main Library — indexes all audio files under the main library folder. Run this after adding new music.
  • Force Rescan — ignores the cache and rescans from scratch.
  • Scan Show Folders / Force Show Rescan — same but for the show library subfolders.
⚠ Always click Save Settings before scanning. The scanner uses whatever folder path is currently saved.

Scheduler tab

Lets you define a daily programme — which show plays when, for how long, and what plays in the gaps.

Top row

  • Scheduler enabled — tick to allow the scheduler to start blocks automatically at their configured times.
  • Start time — the time of day the first block begins (24-hour, HH:MM). The timeline preview updates live as you type.
  • Fill mode — what happens when no show block is scheduled: random plays random tracks from your library, stop goes silent, loop restarts the schedule from block 1.

Building a schedule

1
Select a show folder

Click a folder name in the Show folders list on the left. If the list is empty, click ⟳ Refresh or set and scan the Show library parent folder in the Libraries tab first.

2
Set the hours

Type how many hours this show should play in the Hours box. Decimals are allowed — e.g. 1.5 for 90 minutes.

3
Click Add

The block appears in the Schedule blocks list on the right with its start and end times. Repeat to build a full day. Use ▲ ▼ to reorder blocks and Remove to delete the selected one.

4
Save

Click Save Scheduler Settings. The schedule takes effect immediately — if the current time falls inside a block's window the show starts playing right away.

Random fill folders

By default, fill playback (the gaps between shows) uses your main library. If you want a specific set of folders instead, change Fill source to selected_folders, type each folder path in the box and click Add. Use Remove to delete a selected entry.

Commercials

  • Enable commercials — tick to enable ad breaks. Set the Commercials folder to a folder of audio files, then click Scan Commercial Folder.
  • When to fire breaks — choose one or more triggers: after every show block ends, between shows, after every track, or on a timed per-hour basis.
  • Timed breaks/hr — how many timed breaks to fire each hour (0 = off). The backend spaces them evenly across the hour.
  • Guard rails — minimum gap between breaks, minimum show runtime before the first break, and maximum breaks per show block. All default to 0 (no limit).
  • Spots per break — fixed number of ads, or a random range (set both Min and Max above 0 to use the range).
  • Quiet hours — click any hour cell to toggle it amber. No commercial breaks will fire during amber hours.
  • Playback order — random picks ads randomly. ordered_label plays them in numerical order by filename prefix (e.g. o1-ad.mp3, o2-ad.mp3…).
  • Test Commercial Break — triggers a break immediately using the current spot count setting.

Play buttons

  • Play Schedule Now — starts whichever block should be playing at the current time of day.
  • Play Full Schedule — starts from block 1 and plays the entire schedule in order regardless of the clock.

Hour Chimes tab

Plays an audio chime at the top of every hour.

  • Enable hourly chimes — tick to activate. The chime fires at :00 of each hour.
  • Mode — repeat_strike — generates a chime tone that strikes once per hour of the day (1 strike at 1am, 12 strikes at noon, etc.).
  • Mode — audio_drop — plays a specific audio file instead of the generated tone. You must assign a file to each hour you want a chime.
  • Interrupt instead of pause/resume — when ticked, the chime interrupts the current track and the track does not resume afterwards. When unticked (default), the track is paused, the chime plays, then the track resumes from where it left off.

Chimes folder

The folder the Pi looks in for audio_drop chime files by default. Type the full path on the Pi.

Per-hour audio files

The grid shows all 24 hours. Green = a specific file is assigned. Click any hour cell to open the assignment panel below, type the full file path on the Pi, and the cell turns green. Click Clear to remove an assignment for that hour. If audio_drop mode is selected but an hour has no file assigned, no chime plays for that hour.

Click Test Current Hour Chime to trigger the chime for right now immediately without waiting for the next :00.

Monitor tab

Shows what is playing right now and a visual overview of today's schedule.

  • The Now panel shows a badge — playing, commercial, chime, or idle — with the current track or state.
  • The timeline bar shows 00:00 → 24:00 with show blocks in blue and fill gaps in grey. The green line is the current time.
  • Schedule blocks lists each block with its start time, name, and end time.

Logs tab

Shows the last 80 lines of the backend log. Useful for diagnosing problems — errors, scan results, scheduler events, and commercial break triggers all appear here. Click ⟳ Refresh to fetch the latest lines. The log is also written to backend.out in the program folder on the Pi.

Troubleshooting

  • Offline / can't connect — make sure the backend is running on the Pi (bash start_all.sh or python3 run_radio.py). Check the port — it may have moved to 8766 or higher if 8765 was busy. The correct port is printed in backend.out on the first line after startup.
  • No audio / VLC missing — install VLC on the Pi: sudo apt install -y vlc then pip3 install python-vlc --break-system-packages. Restart the backend.
  • Shows list is empty — make sure the Show library parent folder is set and contains at least one subfolder with audio files. Click Scan Show Folders.
  • Schedule not starting automatically — make sure Scheduler enabled is ticked and settings are saved. The backend will catch up if the Pi was off at the scheduled start time.
  • Commercials not playing — check the Commercials folder is set and scanned. Check the Logs tab for suppression messages — a guard rail (quiet hour, min gap, max breaks) may be blocking the break.
  • Chime not playing — if using audio_drop mode, make sure a file is assigned to the current hour (green cell). Test it with the Test button.
✎ The Logs tab is almost always the fastest way to diagnose any problem. Backend errors, scanner output, and scheduler decisions are all logged there in real time.
